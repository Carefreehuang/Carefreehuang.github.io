Map

HashMap 和 Hashtable 底层都是哈希表（也称散列表），其中维护了一个长度为 2 的幂次方的 Entry 类型的数组 table，数组的每一个索引位置被称为一个桶 (bucket)，你添加的映射关系(key,value)最终都被封装为一个 Map.Entry 类型的对象，放到某个 table[index]桶中。

使用数组的目的是查询和添加的效率高，可以根据索引直接定位到某个table[index]

HashMap中数据的添加

  • JDK7
  • JDK8 相较于 JDK7 的不同之处
    • 使用 HashMap()的构造器创建对象时，并没有在底层初始化长度为 16 的 table 数组
    • jdk8 中添加的 key,value 封装到了 HashMap.Node 类的对象中。而非 jdk7 中的 HashMap.Entry。
    • 索引冲突的时候，jdk8使用的是尾插法，jdk7使用的是头插法。尾插法可以防止在并发处理的时候形成环形链表
    • jdk7 时底层的数据结构是：数组+单向链表。 而 jdk8 时，底层的数据结构是：数组+单向链表+红黑树。
      • 红黑树出现的时机：当某个索引位置 i 上的链表的长度达到 8，且数组的长度超过 64时，此索引位置上的元素要从单向链表改为红黑树。
      • 如果索引 i 位置是红黑树的结构，当不断删除元素的情况下，当前索引 i 位置上的元素的个数低于 6 时，要从红黑树改为单向链表。

  • 解释java中的HashMap底层的table数组长度为什么是2的幂次方，有什么作用，是如何实现的?

    • 长度为2的幂次方的作用
      • 快速计算哈希值的索引：当 table 的长度为2的幂次方时，可以通过使用与运算来加速计算哈希值对应的索引，比模运算提高了性能。
      • 由于数组长度是 2 的 n 次幂，当进行位运算时，哈希值的高位和低位都参与索引计算，这使得元素在数组中的分布更加均匀，减少了哈希冲突的可能性。

  • LinkedHashMap 和 HashMap的区别在哪里？

    • LinkedHashMap使用了双向链表来保持元素的插入顺序
    • LinkedHashMap保证了元素的迭代和插入顺序一致

  • 说说你理解的哈希算法

    • 哈希算法是一种将数据映射到固定长度的哈希值或哈希码的算法。
    • hash 是具有唯一性且不可逆的
      • 唯一性是指相同的“对象”产生的 hash code 永远是一样的。
    • 比较出名的 hash 算法有 MD5、SHA。

  • Entry 中的 hash 属性为什么不直接使用 key 的 hashCode()返回值呢？

    • 为了提高hash code 值与 (table.length-1)的按位与完的结果，尽量的均匀分布。
    • 使用扰动函数，可以将原始的哈希值进行进一步的混合和扰动，以减少碰撞的概率，并使元素在哈希表中更均匀地分布。扰动函数一般会对原始的哈希值进行一系列的位运算、移位和异或等操作，以增加哈希值的随机性，从而减少碰撞的可能性。
    • 思路都是将 hashCode 值的高位二进制与低位二进制值进行了异或，然高位二进制参与到 index 的计算中。
    • 为什么要 hashCode 值的二进制的高位参与到 index 计算呢?
      • 因为一个 HashMap 的 table 数组一般不会特别大，至少在不断扩容前那么table.length-1 的大部分高位都是 0.
      • 直接用 hashCode 和 table.length-1 进行&运算的话，就会导致总是只有最低的几位是有效的。
      • 那么就算你的 hashCode()实现的再好也难以避免发生碰撞，
      • 这时让高位参与进来的意义就体现出来了。它对 hashcode 的低位添加随机性并且混合了高位的部分特征，显著减少了碰撞冲突的发生。
